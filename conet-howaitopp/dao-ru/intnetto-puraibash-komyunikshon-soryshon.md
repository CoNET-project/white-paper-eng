# インターネット プライバシー コミュニケーション ソリューション

ほとんどすべての既存のプライバシー強化ツールは、実際の送信者 IP アドレスのメタデータを隠すために、送信者 IP アドレスをサービス プロバイダーの IP アドレスに置き換えることに基づいて開発していく方法であります。

[**仮想プライベート ネットワークVPN**](https://ja.wikipedia.org/wiki/Virtual\_private\_network)

送信者はまず VPN プロバイダーにアクセスし、VPN プロバイダーから提供された IP アドレスを使用して宛先 IP アドレスにアクセスします。

**利点:** すべてのインターネット アプリケーションとのシームレスな統合。 送信者は実際のアクセス先のIPアドレスを隠すため、監視者は送信者が誰と通信しているのか、どのようなアプリケーションを使用しているのかを知ることを避けることができます。ISPはVPNプロバイダーの IP アドレスを取得するため、送信者の本当の場所を隠すことができます。&#x20;

**欠点:** ユーザーは料金を実名で支払う事により匿名性を保つことができず、ユーザーのすべてがVPNプロバイダーによって記録される可能性があり、VPNは送信者の匿名化のみ、発信元の匿名化を使う場合としてができません。VPNの固有の通信プロトコルがネットワーク監視によってブロックされることも (中国の[グレート ファイアウォール](https://ja.wikipedia.org/wiki/%E3%82%B0%E3%83%AC%E3%83%BC%E3%83%88%E3%83%BB%E3%83%95%E3%82%A1%E3%82%A4%E3%82%A2%E3%82%A6%E3%82%A9%E3%83%BC%E3%83%AB)は VPN 通信プロトコルをシャットダウンします)。

[**トーオニオンネットワーク**](https://ja.wikipedia.org/wiki/Tor)

VPNのアップグレードと言えますが、ユーザー秘密を一つ経路にしたVPNに、複数の[ノード](https://ja.wikipedia.org/wiki/%E3%83%8E%E3%83%BC%E3%83%89\_\(%E3%83%8D%E3%83%83%E3%83%88%E3%83%AF%E3%83%BC%E3%82%AF\))を経由させて分かれて、各部分が互いに共謀しない限り、VPNを超えるプライバシー実現ができます。

**利点:** VPN よりも優れたプライバシー

**欠点:** VPNより多くのノードを経由するため、各ノードで暗号化・復号化の操作が必要となり、VPNよりも通信効率が低くなります。Torノードはすべてボランティアによって提供されているため、通信品質が保証されず、商用アプリケーションは実現できません。特殊な通信プロトコルには監視によって区別され通信がブロックされた可能性があります。 拡張テクノロジーを通じて、機能限定的な ISP 匿名性を提供します。

[**分散型VPN**](https://clearvpn.com/blog/dvpn-vs-vpn/)

ブロックチェーン インセンティブ メカニズムに基づく分散型 VPN サービスです。VPN の長所と短所をすべて継承しますが。 ワークロード プルーフ メカニズムは、参加者が高品質のサービスを提供することを促進します。ワークロード プルーフによりユーザーの匿名化が可能になり、隠れた危険が残されます。 ユーザーは匿名でロイヤルティを支払うことで、より高い匿名性を得ることができます。

[**ハイブリッドネットNYM**](https://nymtech.net/)

ブロックチェーンインセンティブの完全分散型ハイブリッドネットワーク。 Tor と同様のワークロード耐性のあるハイブリッド ネットワークを提供すると、ユーザーはノード IP アドレスをランダムに使用して、ハイブリッド ネットワーク経由ですべてのトラフィックを送信できます。

**利点:** VPN や Tor よりも優れたプライバシー、ブロックチェーン インセンティブ メカニズムによる分散化、ワークロード プルーフ メカニズムにより高品質のサービスが保証されます。

**欠点:** 特殊な通信プロトコルはネットワーク監視によって通信がブロックされた可能性があります。追加のプルーフ・オブ・ワーク・メカニズムにより使用コストが増加します。ユーザーが匿名化される可能性があります。 拡張テクノロジーを通じて、機能限定的な ISP 匿名性を提供します。
